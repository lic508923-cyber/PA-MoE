# PA-MoELog

这是 **PA-MoELog** 第一阶段的最小可运行骨架。PA-MoELog 是一个面向新系统少标签适配的跨系统日志异常检测模型，特点是免解析、保留参数信息。

## 当前包含内容

当前实现聚焦于完整跑通一次前向传播，而不是完整训练流水线。

核心模块：

1. **免解析、保参数的预处理**：不依赖 Drain 或 Spell，抽取 `semantic_text`，并显式保留带类型的参数。
2. **参数感知注意力编码器**：将参数嵌入作为参数偏置注入注意力分数。
3. **多源专家池**：运行多个轻量日志专家，并使用稀疏路由权重进行组合。
4. **DoRA-AdaLoRA 适配接口**：使用冻结的基础线性层、低秩方向更新和幅值缩放，不使用 Adapter 模块。
5. **Top-k 稀疏路由与 GMM 能量边界**：将每个样本路由到 top-k 专家，并用稳定的对角协方差 GMM 估计异常能量。

## 运行演示

在项目根目录执行：

```bash
python -m pa_moelog.demo_forward
```

演示流程：

- 解析四条玩具原始日志。
- 构建 `PAMoELog(num_experts=3, top_k=2, hidden_dim=128)`。
- 执行一次前向传播。
- 打印最终分数、分类器分数、能量分数、路由权重和选中的专家。

## 当前阶段的简化点

- 文本骨干目前是基于哈希的 `nn.Embedding` 编码器，`backbone_name` 预留给后续接入 BERT 或 ModernBERT。
- 参数值目前使用哈希嵌入，而不是训练好的分词器。
- `ExpertPool` 当前会先计算所有专家，再应用稀疏路由权重；后续版本可以只执行被选中的 top-k 专家。
- AdaLoRA 秩预算已经暴露控制器接口，但基于动态秩重要性的重分配仍是 TODO。
- GMM Energy 使用可训练参数和对角协方差，但尚未接入训练目标进行拟合。
- 目前还不包含完整数据集加载器、优化器、训练循环、评估协议和 checkpoint 机制。
